# UE5 第三人称开发项目总结

> 项目：Test_GamePlay  
> 模板：3D 俯视角（TopDown）  
> 引擎：Unreal Engine 5.8

---

## 一、项目概况

基于 UE5 俯视角模板改造的第三人称动作游戏。  
主控角色 Hikari（光），有 WASD 移动、疾跑、近战攻击、受伤死亡系统。  
敌人有自动攻击 AI、生成系统、伤害检测和死亡逻辑。

## 三、玩家系统（HikariPlayerCharacter）

### 3.1 移动（WASD）

| 按键 | 功能 |
|---|---|
| W | 向屏幕上方移动 |
| S | 向屏幕下方移动 |
| A | 向屏幕左方移动 |
| D | 向屏幕右方移动 |
| Shift（按住） | 疾跑（加速） |

移动方向基于**等距视角**计算：
- W = Forward + Right（屏幕右上方向）
- D = Right - Forward（屏幕右下方向）

### 3.2 攻击

**左键（IA_Attack）：** 
1. 播放攻击蒙太奇（AttackMontage，需要在 BP_Hikari 中指定）
2. 同时在角色前方 100cm 生成近战攻击范围（AttackArea）
3. 攻击动画期间不能移动，不能再次攻击

### 3.3 状态机

```
Normal（空闲）→ Attacking（攻击中）→ Normal
Normal → Sprinting（疾跑中）→ Normal
```

- `CanMove()` 只允许 Normal 状态移动
- `CanStartAction()` 只允许 Normal 状态攻击
- Shift 疾跑会中断攻击状态

### 3.4 受伤与死亡

- `HandleDamage(float)` — 扣血，归零时调用 Die()
- `Die()` — 标记死亡、触发 OnPlayerDeath 多播委托（蓝图可绑定）、销毁 Actor
- `bIsDead` — 死亡标记，防止重复触发

## 四、攻击范围系统（AttackAreaBase）

### 4.1 一句话

一个自带碰撞箱的 Actor，碰到敌人就造成伤害。**近战和射弹共用同一个类**，通过参数区分行为。

### 4.2 构造函数创建的组件

```
CollisionBox（根组件，BoxComponent）
```

没有多余组件，蓝图不会出现两个碰撞体。

### 4.3 参数列表

调用 `Initialize()` 传入：

| 参数 | 说明 | 近战（玩家） | 射弹（敌人） |
|---|---|---|---|
| LifeTime | 存在时间（秒） | 0.5s | 3.0s |
| Speed | 移动速度（cm/秒） | 0（原地） | 500+ |
| Damage | 伤害值 | 300 | 60 |
| bEnemyOnly | 是否只伤敌对 | true | true |
| bObstacle | 是否检测墙壁 | false | true |
| bMelee | 是否近战 | true | false |

### 4.4 碰撞通道

| 通道 | 用途 |
|---|---|
| ECC_GameTraceChannel1（DamageArea） | 攻击范围的碰撞体 |
| ECC_GameTraceChannel2（EnemyHitbox） | 敌人的碰撞胶囊体 |
| ECC_GameTraceChannel3（PlayerHitbox） | 玩家的碰撞胶囊体 |

攻击范围同时检测 Channel2 和 Channel3，谁受伤由 `IsValidTarget` 逻辑判断。

### 4.5 命中后的行为

| 类型 | 命中后 |
|---|---|
| 近战（bMelee=true） | 伤害目标，不销毁，等 LifeTime 结束 |
| 远程（bMelee=false） | 伤害目标，0.1s 后销毁 |

### 4.6 障碍物检测

当 `bObstacle = true` 时：  
每帧在飞行方向发射 LineTrace 检测 `ECC_WorldStatic` 物体（墙壁），检测到则销毁。  
适用于射弹（敌人子弹），近战不开启。

## 五、敌人生成系统

### EnemySpawnPoint（生成点）

- 只是个位置标记，带箭头指方向
- 不负责生成逻辑

### DemoRoomManager（房间管理器）

- BeginPlay 时自动收集场景中所有 EnemySpawnPoint
- 延迟 StartDelay 秒后开始生成
- 在每个 SpawnPoint 位置生成 EnemyClass
- 监听敌人死亡，全部击杀后触发 OnRoomClear（蓝图可实现）

## 六、敌人系统（EnemyBase）

### 6.1 结构

```
CapsuleCollision（根组件+碰撞体）
  └── Mesh（网格体，碰撞关闭，仅显示）
```

### 6.2 属性

- `MaxHealth` / `CurrentHealth` — 血量
- `bIsDead` — 死亡标记
- `OnEnemyDeath` — 死亡事件广播
- `AttackAreaClass` — 攻击范围蓝图（需在蓝图中赋值）
- `AttackInterval` — 攻击间隔（默认 5 秒）

### 6.3 行为

- BeginPlay 启动攻击计时器
- 每隔 `AttackInterval` 秒在面前 80cm 生成一次 AttackArea

## 七、蓝图配置指引（新人必看）

### 7.1 BP_Hikari（玩家蓝图）需要赋值的变量

| 分类 | 变量 | 赋值 |
|---|---|---|
| Input | Move Horizontal Action | IA_MoveHorizontal |
| Input | Move Vertical Action | IA_MoveVertical |
| Input | Sprint Action | IA_Sprint |
| Input | Attack Action | IA_Attack |
| Animation | Attack Montage | AM_Hikari_Attack_01 |
| Attack | Attack Area Class | BP_AttackArea（或你创建的子类） |
| Player\|Stats | Max Health | 100（可调） |

### 7.2 BP_EnemyDummy（敌人蓝图）需要赋值的变量

| 分类 | 变量 | 赋值 |
|---|---|---|
| Enemy\|Attack | Attack Area Class | BP_AttackArea（或你创建的子类） |
| Enemy\|Attack | Attack Interval | 5.0（可调） |

### 7.3 DemoRoomManager 场景布置

1. 在场景中放置一个 `BP_DemoRoomManager`
2. 在其细节面板的 `Room|Spawn` → `Enemy Class` 选择 `BP_EnemyDummy`
3. 在场景中放置若干个 `BP_EnemySpawnPoint`
4. 运行游戏，敌人会自动在生成点出现
